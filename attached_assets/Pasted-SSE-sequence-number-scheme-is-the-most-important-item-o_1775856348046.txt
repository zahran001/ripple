One sentence in the architecture doc prevents a subtle correctness bug in Phase 4.



Severity scoring formula is also correct. "Weighted function of depth and criticality" is not a formula, it's a description of a formula. Without the actual expression, Phase 3 tests can only assert set membership, not score values, and the DegradationPlanner is sorting on an undefined basis.
