# Ripple — coding standards

## Subscriber interface contract

Every `Subscriber` implementation must declare its own `highWaterMark()`. This is not a shared constant — it reflects what missing an event actually means for that subscriber:

```java
public interface Subscriber {
    void onEvent(FailureEvent event);

    /**
     * Maximum number of events this subscriber can lag behind the producer
     * before the BackpressureMonitor begins shedding events for it.
     *
     * Implementations that cannot tolerate missing events (AlertRouter,
     * StateStore) must return Long.MAX_VALUE. Implementations with a
     * recovery path (SseFeed) should return a low value to shed early
     * and recover fast via snapshot fetch.
     */
    long highWaterMark();
}
```

Never set `highWaterMark()` to the same value across all implementations without explicitly justifying it. The whole point is that different subscribers have different tolerances.

## Package conventions

Root package: `com.ripple`

Sub-packages follow the architecture layers exactly:

```
com.ripple.engine.probe
com.ripple.engine.topology
com.ripple.engine.blast
com.ripple.engine.stream
com.ripple.api
com.ripple.quorum
com.ripple.model
com.ripple.config
```

Never create a `util`, `helper`, or `common` package. If something doesn't fit the layers above, it belongs in `model` (if it's a value) or reconsider the design.

## Naming conventions

| Element | Convention | Example |
|---|---|---|
| Classes | PascalCase, noun | `CircuitBreaker`, `TopologyGraph` |
| Interfaces | PascalCase, noun (no `I` prefix) | `Prober`, `Subscriber` |
| Records | PascalCase, noun | `ProbeResult`, `GraphSnapshot` |
| Enums | PascalCase, noun | `CircuitBreakerState`, `ProbeStatus` |
| Enum values | SCREAMING_SNAKE_CASE | `CLOSED`, `OPEN`, `HALF_OPEN` |
| Methods | camelCase, verb | `computeBlastRadius()`, `registerService()` |
| Fields | camelCase | `stampedLock`, `highWaterMark` |
| Constants | SCREAMING_SNAKE_CASE | `DEFAULT_PROBE_INTERVAL_MS` |
| Test classes | Suffix `Test` | `CircuitBreakerTest` |
| Test methods | `should_doX_when_Y` | `should_openCircuit_when_failureThresholdExceeded` |

## Value objects — prefer records

Use Java records for all immutable value objects. Records are the default for anything that crosses layer boundaries.

```java
// Preferred
public record ProbeResult(
    ServiceId serviceId,
    ProbeStatus status,
    Duration latency,
    Instant timestamp,
    String detail
) {}

// Only use a class if you need mutability or complex construction logic
```

## Concurrency documentation

Every class that has concurrency semantics must document them in the class-level Javadoc.

```java
/**
 * Thread-safe topology graph backed by a ConcurrentHashMap adjacency list.
 *
 * <p>Reads use StampedLock optimistic reads with fallback to read lock.
 * Structural mutations (add/remove node or edge) acquire the write lock.
 *
 * <p>All public methods are safe to call concurrently from multiple threads.
 * GraphSnapshot instances returned by {@link #snapshot()} are immutable.
 */
public class TopologyGraph { ... }
```

Every field that is guarded by a lock must say so in its field Javadoc or inline comment:

```java
// guarded by stampedLock write lock
private final Map<ServiceId, Set<ServiceId>> adjacency = new ConcurrentHashMap<>();
```

## Exception handling

No swallowed exceptions. Every catch block must do one of:

1. Re-throw as a typed domain exception with context added.
2. Log at the appropriate level and emit a failure metric.
3. Both.

```java
// BAD
try {
    prober.probe(node);
} catch (Exception e) {
    // silently swallowed
}

// GOOD
try {
    prober.probe(node);
} catch (IOException e) {
    log.warn("Probe failed for service {}: {}", node.id(), e.getMessage());
    metrics.increment("probe.failure", "service", node.id().value());
    return ProbeResult.failure(node.id(), e.getMessage());
}
```

## Configuration

Zero hard-coded timeouts, thresholds, or intervals anywhere in production code. Everything goes in `application.yml` and is bound via a `@ConfigurationProperties` class.

```yaml
ripple:
  probe:
    concurrency-limit: 50
    default-interval-ms: 5000
    http-timeout-ms: 2000
  circuit-breaker:
    failure-threshold: 5
    cooldown-ms: 30000
  stream:
    high-water-mark: 10000
  rate-limit:
    requests-per-second: 100
```

```java
@ConfigurationProperties(prefix = "ripple.probe")
public record ProbeProperties(
    int concurrencyLimit,
    long defaultIntervalMs,
    long httpTimeoutMs
) {}
```

## Testing conventions

### Unit tests

- One test class per production class, co-located in `src/test/java` mirroring the main package structure.
- Use Awaitility for all async assertions — never `Thread.sleep()`.
- Use Mockito for stubbing collaborators, never for the class under test.
- Test method names: `should_doX_when_Y`.

```java
@Test
void should_openCircuit_when_failureThresholdExceeded() {
    // arrange — no-op callback; we're testing state, not downstream wiring
    var breaker = new CircuitBreaker(CircuitBreakerConfig.defaults(), state -> {});
    
    // act
    IntStream.range(0, 5).forEach(_ -> breaker.recordFailure());
    
    // assert
    assertThat(breaker.state()).isEqualTo(CircuitBreakerState.OPEN);
}

@Test
void should_transitionToHalfOpen_when_cooldownElapses() {
    var breaker = new CircuitBreaker(CircuitBreakerConfig.withCooldown(Duration.ofMillis(100)), state -> {});
    IntStream.range(0, 5).forEach(_ -> breaker.recordFailure());

    await().atMost(500, MILLISECONDS).until(
        () -> breaker.state() == CircuitBreakerState.HALF_OPEN
    );
}

@Test
void should_invokeCallback_when_transitioningToOpen() {
    List<CircuitBreakerState> observed = new CopyOnWriteArrayList<>();
    var breaker = new CircuitBreaker(CircuitBreakerConfig.defaults(), observed::add);

    IntStream.range(0, 5).forEach(_ -> breaker.recordFailure());

    assertThat(observed).containsExactly(CircuitBreakerState.OPEN);
}
```

### Integration tests

- Extend a `BaseIntegrationTest` that starts Testcontainers (Postgres, Redis) once per test class via `@Container` with `@ClassRule` semantics.
- Integration tests live in `src/test/java` with a `IT` suffix: `TopologyGraphIT`.
- Integration tests are excluded from the standard `test` phase and run in a separate Maven profile: `mvn test -P integration`.

### Architecture tests (ArchUnit)

ArchUnit rules run as standard unit tests in the normal `test` phase — no separate profile needed. They enforce structural constraints that cannot be expressed in the type system.

The mandatory rule for `TopologyGraph` — enforces the `StampedLock` public/private reentrancy discipline:

```java
@AnalyzeClasses(packages = "com.ripple")
class ArchitectureTest {

    @ArchTest
    static final ArchRule topology_graph_public_methods_must_not_call_other_public_methods =
        noMethods()
            .that().arePublic()
            .and().areDeclaredIn(TopologyGraph.class)
            .should().callMethodWhere(
                target().isPublic().and().areDeclaredIn(TopologyGraph.class)
            )
            .because("StampedLock is not reentrant — a public method calling " +
                     "another public method on the same instance will deadlock");
}
```

This rule makes the `StampedLock` reentrancy constraint a failing CI test rather than a code review hope. Add a new `ArchitectureTest` class in `src/test/java/com/ripple` and accumulate all ArchUnit rules there as `@ArchTest` fields.

## Logging

Use SLF4J with Logback. Log levels:

| Level | When to use |
|---|---|
| `ERROR` | Unexpected system failure requiring human attention |
| `WARN` | Expected failure that Ripple handles (probe timeout, circuit opening) |
| `INFO` | Significant lifecycle events (service registered, phase started) |
| `DEBUG` | Per-probe results, per-event fan-out (not in production) |
| `TRACE` | Lock acquisition, BFS steps (never in production) |

Always use structured logging parameters, never string concatenation:

```java
// BAD
log.info("Probe failed for " + nodeId + " after " + latency.toMillis() + "ms");

// GOOD
log.warn("Probe failed for service {} after {}ms", nodeId, latency.toMillis());
```

## Code generation rules for Claude

When Claude generates code for this project:

- Include the full file: package declaration, all imports, class, Javadoc.
- Generate the unit test class alongside every new production class.
- Use `var` for local variables where the type is obvious from the right-hand side.
- Prefer `record` for value objects unless mutability is explicitly required.
- Never use `Thread.sleep()` — use Awaitility in tests, `ScheduledExecutorService` or `Flux.interval()` in production.
- Annotate all Spring beans with the narrowest applicable annotation (`@Service`, `@Repository`, `@Component`).
- Never import `*`.
- `CircuitBreaker` always takes a `Consumer<CircuitBreakerState> onStateChange` as its second constructor argument. Unit tests that are not testing the callback pass a no-op: `state -> {}`.
- `SystemReadModel` and `RedisSchema` live in `com.ripple.model`. Never inline Redis key strings in production code — always use `RedisSchema` static methods.
- Never create a package named `util`, `helper`, `common`, or `persistence`. Shared contracts and value objects go in `com.ripple.model`.
